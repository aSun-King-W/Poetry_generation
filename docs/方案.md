# 古诗生成系统 - 实现方案

> **任务：** 输入古诗上句 → 模型自动生成对应下句
> 
> **三种架构：**
>   - 方法一：Decoder-Only Transformer（从零实现）
>   - 方法二：Encoder-Decoder Transformer（从零实现）
>   - 方法三：Pretrained Model Fine-Tuning（`uer/gpt2-chinese-poem` 微调）
> 
> **资源：** GPU (CUDA) | Gradio Web Demo | LaTeX 英文报告
> 
> **数据集：** `chinese-poetry/chinese-poetry`（GitHub 原始仓库，唐诗宋诗 JSON）

## 数据处理

### 数据源
- 克隆 `chinese-poetry/chinese-poetry` GitHub 仓库到 `data/raw/`
- 解析 `json/` 目录下唐诗和宋诗的 JSON 文件

### 数据清洗与配对
1. 遍历所有 JSON 文件，提取每条记录的 `content` 字段
2. 筛选长度为 5 或 7 个字的诗句（五言/七言）
3. 同一诗中相邻行配对：第 0-1 行、第 2-3 行……
4. 过滤异常数据

### 词表构建
- 统计字符频率，保留频率 ≥ 3 的字符
- 按频率降序排列，构建字符到索引的映射
- 特殊标记：`<PAD>`(0), `<UNK>`(1), `<BOS>`(2), `<EOS>`(3), `<SEP>`(4)
- 词表大小约 5000-8000，保存为 `data/vocab.json`

### 各模型数据格式
- **Decoder-Only**：输入拼接为 "上句 SEP 下句 EOS"，因果掩码预测下一个 token
- **Encoder-Decoder**：Encoder 输入 "上句"，Decoder 输入 "BOS 下句"，标签 "下句 EOS"
- **Pretrained**：使用预训练 tokenizer（`[CLS] 上 句 字 间 空 格 [SEP] 下 句`）

### 数据集划分
- train / valid / test = 8 : 1 : 1
- 序列填充/截断到固定长度（32-64）

## Transformer 核心组件（从零实现）

所有组件在 `src/models/transformer.py` 中实现。使用 PyTorch 基本张量运算实现，不调用 `torch.nn.Transformer` 等预建 API。

### 1. MultiHeadAttention
- 缩放点积注意力：`Attention(Q,K,V) = softmax(QK^T / √d_k) V`
- 支持 causal mask（下三角矩阵，防止 Decoder 看到未来 token）
- 支持 cross-attention（K/V 来自 Encoder 输出）
- 8 个头，每个头维度 d_k = d_model / n_heads
- 输入/输出形状：`(batch, seq_len, d_model)`

### 2. PositionalEncoding
- 正弦/余弦编码，无需学习参数
- `PE(pos, 2i) = sin(pos / 10000^(2i/d_model))`
- `PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))`
- 支持最大长度 256

### 3. FeedForward
- 两层全连接：d_model → d_ff → d_model（d_ff = 4 × d_model）
- 激活函数：GELU
- 含 Dropout

### 4. LayerNorm
- 可学习参数 γ（缩放）和 β（偏移）
- ε = 1e-5

### 5. TransformerBlock（Decoder 块）
- 子层 1：Masked Multi-Head Self-Attention
- 子层 2：Cross-Attention（可选，用于 Encoder-Decoder 架构）
- 子层 3：FeedForward
- 每个子层后：Residual Connection + LayerNorm（Pre-Norm）

### 6. EncoderBlock（Encoder 块）
- 子层 1：Multi-Head Self-Attention（无 mask）
- 子层 2：FeedForward
- 每个子层后：Residual Connection + LayerNorm

## 方法一：Decoder-Only Transformer

### 模型架构
```
TokenEmbedding → PositionalEncoding → TransformerBlock×6 → Linear(vocab_size)
```
- N = 6 层，d_model = 256，n_heads = 8
- 因果掩码（下三角矩阵 + PAD 掩码）

### 训练

| 超参数 | 取值 |
|--------|------|
| 嵌入维度 d_model | 256 |
| 层数 | 6 |
| 注意力头 | 8 |
| FeedForward 维度 | 1024 |
| Dropout | 0.1 |
| 学习率 | 1e-4 |
| Batch Size | 64 |
| 最大序列长度 | 32 |
| Epochs | 30 |
| 标签平滑 | ε = 0.1 |

- **优化器**：Adam (β₁=0.9, β₂=0.999, ε=1e-8)
- **学习率调度**：warmup 4000 步 + cosine decay
- **梯度裁剪**：max_norm = 1.0
- **损失**：CrossEntropyLoss（忽略 PAD 位置）

### 生成
- 输入上句 → 拼接 SEP → 自回归逐 token 生成 → 遇到 EOS 或达到最大长度停止
- 支持贪心解码、集束搜索（beam=5）、温度采样（t=0.7-1.2）

## 方法二：Encoder-Decoder Transformer

### 模型架构
- **Encoder**（4 层）：Token嵌入 + PE → EncoderBlock×4 → 编码向量
- **Decoder**（4 层）：Token嵌入 + PE → TransformerBlock×4（含 Cross-Attn）→ Linear(vocab_size)
- Encoder 输出作为 Decoder Cross-Attention 的 K、V

### 训练

| 超参数 | 取值 |
|--------|------|
| d_model | 256 |
| Encoder 层数 | 4 |
| Decoder 层数 | 4 |
| 注意力头 | 8 |
| d_ff | 1024 |
| Dropout | 0.1 |
| 学习率 | 1e-4 |
| Batch Size | 64 |
| Epochs | 30 |

**数据格式：** Encoder 输入"上句"，Decoder 输入"BOS 下句"，标签"下句 EOS"

### 生成
- Encoder 前向一次获得编码向量
- Decoder 自回归解码（逐 token 生成）

## 方法三：Pretrained Fine-Tuning

### 预训练模型
- 使用 `uer/gpt2-chinese-poem`（基于 GPT-2 架构，80 万首古诗预训练）
- Tokenizer：`BertTokenizer.from_pretrained("uer/gpt2-chinese-poem")`
- 字间空格 + [CLS] 前缀

### 微调训练

| 超参数 | 取值 |
|--------|------|
| 学习率 | 2e-5 |
| Batch Size | 16 |
| 最大序列长度 | 64 |
| Epochs | 10 |
| 优化器 | AdamW |
| 权重衰减 | 0.01 |

### 生成
- 使用预训练模型已有能力 + 微调后的诗歌配对知识
- 可对比微调前 vs 微调后的生成效果

## 生成策略（`src/generate.py`）

三种解码方式：
1. **贪心解码** — 每步选概率最高的 token，速度快
2. **集束搜索** — beam_size = 5，保留 top-5 候选路径，质量更好
3. **温度采样** — temperature 可调（0.7-1.2），控制多样性

## 评估方案

| 指标 | 说明 |
|------|------|
| **Perplexity (PPL)** | 模型对测试集的困惑度，越低越好 |
| **BLEU-4** | 生成下句与参考下句的 4-gram 重合度 |
| **人工评估** | 随机抽取 50 个案例，评判语义连贯性和格律匹配度 |

评估脚本：`src/evaluate.py`，加载训练好的 checkpoint，计算测试集指标。

## Gradio Web Demo（`src/app.py`）

- **输入区：** 文本框输入古诗上句
- **模型选择：** 三个模型可单选或全选对比
- **解码策略选择：** 贪心 / 集束搜索 / 温度采样
- **生成按钮：** 点击生成
- **输出区：** 显示各模型生成结果，三模型并排对比

启动方式：`python src/app.py`

## 验证方式

1. 每个模型训练完成后，输入测试上句观察生成结果是否合理
2. 计算测试集 Perplexity 和 BLEU-4 分数
3. Gradio Web 界面手动测试多组输入
4. 对比三个模型在同一输入下的生成效果差异
5. 检查代码能否从零正常运行，确保可复现
