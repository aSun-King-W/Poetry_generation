# 古诗生成系统 - 实现方案

> **任务：** 输入古诗上句 → 模型自动生成对应下句
> 
> **三种架构：**
>   - 方法一：Decoder-Only Transformer（从零实现）
>   - 方法二：Encoder-Decoder Transformer（从零实现）
>   - 方法三：Pretrained Model Fine-Tuning（`uer/gpt2-chinese-poem` 微调）
> 
> **资源：** GPU (CUDA) | Gradio Web Demo | LaTeX 英文报告
> 
> **数据集：** `chinese-poetry/chinese-poetry`（GitHub 原始仓库，唐诗宋诗 JSON）

## 阶段二：数据预处理

**目标：** 清洗数据、构建诗句对、建立词表、划分数据集

### 2.1 数据提取与清洗 (`utils/tokenizer.py`)

1. 遍历 `data/raw/chinese-poetry/json/` 下所有 JSON 文件（唐诗、宋诗）
2. 从每条记录的 `content` 字段提取诗句行
3. 筛选长度为 5 或 7 个字的诗句（五言/七言）
4. 同一首诗中相邻行配对为 (上句, 下句)：第 0-1 行一对、第 2-3 行一对……
   - 绝句 4 行 → 2 对，律诗 8 行 → 4 对
5. 过滤过短或过长的异常数据

### 2.2 构建词表

1. 遍历所有诗句，统计字符频率
2. 保留频率 ≥ 3 的字符，按频率降序排列
3. 添加特殊标记：`<PAD>`(0), `<UNK>`(1), `<BOS>`(2), `<EOS>`(3), `<SEP>`(4)
4. 词表大小预计 5000-8000
5. 保存为 `data/vocab.json`

### 2.3 构建数据集 (`utils/dataset.py`)

1. 实现 PyTorch Dataset，为三种模型提供不同数据格式：
   - **Decoder-Only**：输入 "上句 SEP 下句 EOS"，训练时用因果掩码
   - **Encoder-Decoder**：Encoder 输入 "上句"，Decoder 输入 "BOS 下句"，标签 "下句 EOS"
   - **Pretrained**：使用预训练 tokenizer（字间空格 + [CLS] 前缀）
2. 序列填充/截断到固定长度（32-64）
3. 划分 train/valid/test = 8:1:1
4. 保存处理后的数据到 `data/processed/`

## 阶段三：Transformer 核心组件（从零实现）

**目标：** 从零实现 Transformer 的所有核心构建块，供方法一和方法二共用

> **约束：** 使用 PyTorch 基本张量运算，不调用 `torch.nn.Transformer`、`torch.nn.MultiheadAttention` 等预建 Transformer API

在 `models/transformer.py` 中实现以下组件：

### 3.1 `MultiHeadAttention`
- 缩放点积注意力（Scaled Dot-Product Attention）：`Attention(Q,K,V) = softmax(QK^T / √d_k) V`
- 支持 causal mask（下三角矩阵，防止看到未来 token）
- 支持 cross-attention（K/V 来自 Encoder 输出）
- 输入输出形状：`(batch, seq_len, d_model)`
- 8 个注意力头，每个头维度 d_k = d_model / n_heads

### 3.2 `PositionalEncoding`
- 正弦/余弦编码（Sinusoidal Positional Encoding），无需学习参数
- `PE(pos, 2i) = sin(pos / 10000^(2i/d_model))`
- `PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))`
- 支持最大长度 256

### 3.3 `FeedForward`
- 两层全连接：`d_model → d_ff → d_model`（d_ff = 4 * d_model = 1024）
- 激活函数：GELU
- 含 Dropout

### 3.4 `LayerNorm`
- 含可学习参数 γ（缩放）和 β（偏移）
- epsilon = 1e-5

### 3.5 `TransformerBlock`（Decoder 块）
- 子层 1：Masked Multi-Head Self-Attention
- 子层 2：Cross-Attention（可选，用于 Encoder-Decoder）
- 子层 3：FeedForward
- 每个子层后接：Residual Connection + LayerNorm（Pre-Norm 架构）

### 3.6 `EncoderBlock`（Encoder 块）
- 子层 1：Multi-Head Self-Attention（无 mask）
- 子层 2：FeedForward
- 每个子层后接：Residual Connection + LayerNorm

## 阶段四：Decoder-Only Transformer（方法一）

**目标：** 实现 Decoder-Only 架构，完成训练和生成

### 4.1 模型定义 (`models/decoder_only.py`)

```
TokenEmbedding → PositionalEncoding → TransformerBlock×N → Linear(d_model → vocab_size)
```

- N = 6 层，d_model = 256，n_heads = 8
- 因果掩码（下三角矩阵 + PAD 掩码）

### 4.2 训练 (`train.py` 中的 Decoder-Only 分支)

**数据格式：** 输入句 = "上句 SEP 下句 EOS"，标签 = 输入右移一位

| 超参数 | 取值 |
|--------|------|
| 嵌入维度 d_model | 256 |
| 层数 N | 6 |
| 注意力头 n_heads | 8 |
| FeedForward 维度 d_ff | 1024 |
| Dropout | 0.1 |
| 学习率 | 1e-4 |
| Batch Size | 64 |
| 最大序列长度 | 32 |
| Epochs | 30 |
| 标签平滑 | ε = 0.1 |

**优化器：** Adam (β₁=0.9, β₂=0.999, ε=1e-8)
**学习率调度：** warmup 4000 步 + cosine decay
**梯度裁剪：** max_norm = 1.0
**损失：** CrossEntropyLoss（忽略 PAD 位置）
**保存：** 每轮根据 valid loss 保存最佳 checkpoint

### 4.3 生成 (`generate.py`)

三种解码策略：

1. **贪心解码：** 每步选概率最高的 token
2. **集束搜索（Beam Search）：** beam_size=5，保留 top-5 候选路径
3. **温度采样：** temperature 可调（0.7-1.2）

生成流程：输入上句 → 拼接 SEP → 自回归逐 token 生成 → 遇到 EOS 或达到最大长度停止

## 阶段五：Encoder-Decoder Transformer（方法二）

**目标：** 实现 Encoder-Decoder 架构，完成训练和生成

### 5.1 模型定义 (`models/encoder_decoder.py`)

**Encoder：** TokenEmbedding → PositionalEncoding → EncoderBlock×N → 编码向量
**Decoder：** TokenEmbedding → PositionalEncoding → TransformerBlock×N（含 Cross-Attn）→ Linear(d_model → vocab_size)

- Encoder N=4 层，Decoder N=4 层，d_model=256，n_heads=8
- Encoder 输出作为 Decoder Cross-Attention 的 K、V

### 5.2 训练

**数据格式：**
- Encoder 输入：上句
- Decoder 输入：BOS + 下句
- 标签：下句 + EOS

| 超参数 | 取值 |
|--------|------|
| 嵌入维度 d_model | 256 |
| Encoder 层数 | 4 |
| Decoder 层数 | 4 |
| 注意力头 n_heads | 8 |
| FeedForward 维度 d_ff | 1024 |
| Dropout | 0.1 |
| 学习率 | 1e-4 |
| Batch Size | 64 |
| 最大序列长度 | 32 |
| Epochs | 30 |

其余（优化器、调度、裁剪、损失）同 Decoder-Only

### 5.3 生成

- Encoder 前向一次，获得编码向量
- Decoder 自回归解码（与 Decoder-Only 类似，但多出 Cross-Attention 步骤）

## 阶段六：Pretrained Fine-Tuning（方法三）

**目标：** 加载预训练模型并在诗歌配对数据上微调

### 6.1 模型加载 (`models/pretrained.py`)

- 加载 `uer/gpt2-chinese-poem`（基于 GPT-2 架构，80 万首古诗预训练）
- 使用 `BertTokenizer.from_pretrained("uer/gpt2-chinese-poem")`
- **注意：** 该 tokenizer 需要在字之间加空格，且加 [CLS] 前缀

### 6.2 微调训练

**数据格式：** `[CLS] 上 句 字 间 空 格 [SEP] 下 句 字 间 空 格`

| 超参数 | 取值 |
|--------|------|
| 学习率 | 2e-5（预训练模型用小学习率） |
| Batch Size | 16（模型参数量大） |
| 最大序列长度 | 64 |
| Epochs | 10 |
| 优化器 | AdamW |
| 权重衰减 | 0.01 |

### 6.3 生成

- 基于预训练模型的生成能力 + 微调后的诗歌配对知识
- 可对比训练前和训练后的生成效果

## 阶段七：评估方案

**目标：** 定量和定性评估三种模型的生成效果

| 指标 | 说明 |
|------|------|
| **Perplexity (PPL)** | 模型对测试集的困惑度，越低越好 |
| **BLEU-4** | 生成下句与参考下句的 4-gram 重合度 |
| **人工评估** | 随机抽取 50 个案例，评判语义连贯性和格律匹配度 |

评估脚本：`evaluate.py`，加载训练好的 checkpoint，计算测试集指标

---

## 阶段八：Gradio Web Demo

**目标：** 构建交互式 Web 界面，展示三个模型的生成效果

### `app.py` 功能

- **输入区：** 文本框输入古诗上句
- **模型选择：** 三个模型可单选或全选做对比
- **解码策略：** 贪心 / 集束搜索（beam=5）/ 温度采样（t=0.8）
- **生成按钮：** 点击生成
- **输出区：** 显示各模型生成的下句，三模型并排对比
- **异常处理：** 空输入、特殊字符等

启动方式：`python app.py`
